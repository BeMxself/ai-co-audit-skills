# AI Co-Audit Skills — Implementation Audit

[English](README.en.md) | [中文](README.md)

A unified implementation-audit skill and automation framework supporting `claude`, `codex`, `kiro-cli`, and custom runners.

## Installation

### Claude Code (Plugin)

Add this repo as a marketplace:

```text
/plugin marketplace add BeMxself/ai-co-audit-skills
```

Install plugin:

```text
/plugin install implementation-audit@BeMxself-ai-co-audit-skills
```

Then invoke `/implementation-audit`.

### Codex (Skills)

Option A (recommended):

```text
$skill-installer Please install this skill from GitHub repo BeMxself/ai-co-audit-skills:
- skills/implementation-audit
```

Option B (manual):

```bash
mkdir -p ~/.agents/skills
rsync -a skills/implementation-audit ~/.agents/skills/
```

## Runner Model

The workflow uses two roles:

- `reviewer`
- `reviser` (merge phase is executed by reviser)

Defaults:
- `reviewer=codex`
- `reviser=claude`

Set at init time:

```bash
bash automation/implementation-audit/init.sh \
  --input docs/plans/2026-02-11-security-external-auth-design.md \
  --reviewer codex \
  --reviser kiro
```

Override at run time:

```bash
./.ai-workflows/<task-id>/run --reviewer codex --reviser kiro
```

## Kiro Setup

When using `kiro` runner, create/update an agent first:

```bash
bash automation/implementation-audit/setup-kiro-agent.sh \
  --project-root "$(pwd)" \
  --agent-name ai-co-audit-kiro-opus \
  --model claude-opus-4.6
```

`run` performs a preflight tool check for Kiro agent file read/write capability and fails fast if missing.

## Quick Start

Initialize:

```bash
bash automation/implementation-audit/init.sh \
  --input docs/plans/2026-02-11-security-external-auth-design.md
```

